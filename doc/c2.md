# C2 : Resources

A API disponibiliza 6 recursos principais: Restaurantes, Pratos, Clientes, Ingredientes, Entregas e Pedidos; cada um com as respetivas operações CRUD essenciais: post, get, put e delete.

## 1 - Restaurantes
Faz a gestão dos restaurantes do sistema.

**Operações CRUD:**

- GET /api/restaurantes -> 
- GET /api/restaurantes/{id}
- POST /api/restaurantes
- PUT /api/restaurantes/{id}
- DELETE /api/restaurantes/{id}

**Campos:**

- 'nome' (obrigatório)
- ...

## 2 - Pratos
Faz gestão dos pratos pedidos dos restaurantes.

**Operações CRUD:**

- GET /api/pratos
- GET /api/pratos/{id}
- POST /api/pratos
- PUT /api/pratos/{id}
- DELETE /api/pratos/{id}

## Relação entre os Recursos

**Relações 1-n**
- **restaurantes -> pratos**: Um restaurante tem vários pratos
- **clientes -> pedidos**: Um cliente pode fazer vários pedidos
- **restaurantes -> pedidos**: Um restaurante recebe vários pedidos

**Relação m-n**
- **pratos <-> ingredientes**: Um prato pode ter vários ingredientes, um ingrediente pode estar em vários pratos (via tabela `pratos_ingredientes`)
